# 海外华人文化贺卡 — MVP 说明（跑通闭环）

> 目标：用**最少功能**验证「选卡 → 编辑 → 生成 → 分享」全流程可跑通。

---

## 1. MVP 定义

**一句话**：用户选一张预设主题贺卡，改几个字（或用有限次 AI 生成祝福语），生成一张图或一页 H5，能保存或复制链接发给亲友。

**刻意不做（本版排除）**

- 礼品支付、红包、第三方电商跳转  
- 复杂裂变（多级邀请、打开计数领奖）— 可后续再加  
- 小程序 / App — 优先 **网页（H5）** 或 **PWA**，降低上架成本  
- 用户上传图片、UGC 模板库  

---

## 2. 唯一主流程（必须全部打通）

```
进入首页
  → 选择分类（如：春节 / 中秋 / 某个节气）
  → 选一张模板
  → 编辑：署名 + 祝福语（可选手填 / AI 生成）
  → 预览
  → 生成成品（图片下载 或 可分享的公开链接）
  → 结束（用户把图或链接发微信等）
```

**验收标准**：从陌生访客到拿到可分享的成品，**全程无人工介入**、无阻断报错。

---

## 3. 功能清单（只做这些）

| 序号 | 功能 | 说明 |
|------|------|------|
| F1 | 模板列表 | 先做 **5～8 张** 静态模板即可（设计好背景 + 预留文字区） |
| F2 | 分类 | **2～3 个**分类足够，例如：春节、中秋、清明/端午（任选） |
| F3 | 在线编辑 | 改：称呼/署名、正文祝福语；字数上限避免排版炸版 |
| F4 | AI 祝福语（可选但建议有） | 选语气（如：给长辈 / 给朋友）→ 调 API 生成 1～3 条供选用；**每用户每日 N 次**（如 3 次），超出提示明天再来或手填 |
| F5 | 生成成品 | **服务端合成**：模板底图 + 文字 → 输出 **PNG/JPG**；或前端 canvas 导出（MVP 可先用简单方案） |
| F6 | 分享 | **下载图片** + **复制一句分享文案**（可选）；若有精力再做 **短链打开的同一张贺卡展示页**（只读） |

---

## 4. 用户与次数（MVP 极简版）

| 项目 | 建议 |
|------|------|
| 登录 | **可不登录**：用浏览器 localStorage 记设备级「今日 AI 已用次数」；或 **邮箱/手机号极简注册** 便于以后扩展（二选一，先定一个） |
| AI 次数 | 例如每设备每日 **3 次**；用完提示清晰，不弹付费（MVP 无支付） |
| 数据 | 若做「分享链接」：保存 `贺卡实例 ID + 文案字段` 到数据库，链接只读展示 |

---

## 5. 内容与模板（最小工作量）

- 文案：每个分类配 **1 段默认祝福语** + **1 句可选古诗**（用公有领域原文 + 标注出处）。  
- 视觉：外包或自用 Figma 出图，固定安全区放字，**不要** MVP 里做拖拽排版编辑器。

---

## 6. 技术形态（推荐最小组合）

| 层 | 建议 |
|----|------|
| 前端 | 单页应用（React / Vue 均可）+ 移动端适配宽度 |
| 后端 | 任意熟悉栈：Node / Python，提供：模板列表、保存贺卡、合成图接口（若合成在后端）、可选短链 GET |
| 存储 | 模板图片放对象存储或静态目录；贺卡记录用 SQLite/PostgreSQL 均可 |
| AI | 调用一家大模型 API，**服务端**转发，密钥不放前端 |

**合成方案二选一（先跑通再优化）**

- **A**：前端 Canvas 把文字画在底图上 → `toDataURL` 下载（实现快，字体要处理好）。  
- **B**：后端 Sharp/Pillow 等拼图（字体、清晰度更稳，略多工作量）。

---

## 7. 2～3 周节奏（参考）

| 周次 | 产出 |
|------|------|
| 第 1 周 | 静态模板 + 首页/列表/编辑页 UI；手填祝福语可预览 |
| 第 2 周 | 合成下载跑通；接 AI；限次逻辑 |
| 第 3 周 | 可选：只读分享页 + 简单埋点；修移动端与边界情况 |

---

## 8. 上线前自检清单

- [ ] 手机浏览器上从选模板到下载成功  
- [ ] AI 失败时有默认文案兜底，不白屏  
- [ ] 长祝福语截断或换行正常  
- [ ] 隐私：若存分享页，链接勿暴露可枚举 ID 的用户隐私（可用随机 UUID）  

---

## 9. MVP 之后的自然延伸（本文档不实现）

邀请得 AI 次数、节气 push、订阅去水印、礼品跳转 —— 等闭环验证有人用后再加。
